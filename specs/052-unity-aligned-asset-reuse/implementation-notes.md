## Plan Review Fix Loop

- 2026-06-17: `/plan-review` found P1 freshness and rollback gaps in the Unity-aligned texture reuse slice: missing or zero texture mapping candidates did not always produce a mapping freshness dependency, auto-imported texture side effects could survive if the later model import failed, and texture path/name matching was not consistently case-insensitive across Windows/macOS-style workflows.
- 2026-06-17: Fixed those P1s by moving model texture mapping dependency value/fingerprint generation into `ModelTextureReferenceResolver`, recording mapping dependencies even for unresolved/empty candidate sets, tracking and rolling back auto-created texture `.meta` files plus committed texture artifact roots when model import fails, and making source-path/name-search matching case-insensitive in both resolver/importer paths.
- 2026-06-17: `/plan-review` also found that explicit remap resolution must prefer stable target GUIDs over stored editor paths, otherwise a moved/renamed texture could break a Unity-style remap. The importer now resolves remap targets by `targetAssetId` first and uses `targetEditorPath` only as a fallback/display path.
- 2026-06-17: Added AssetDatabase currentness support for virtual `PathToGuidMapping` dependency values of the form `project|<query>|name-search/source-path`, recomputing the candidate fingerprint from the refreshed source database and persisted texture manifests instead of treating the value as a physical meta path.
