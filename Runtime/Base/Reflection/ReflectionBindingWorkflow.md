# Reflection Binding Workflow

## Current status

Nullus now uses `Tools/MetaParser` as the only maintained reflection code generator.

- Runtime modules generate reflection glue into `Runtime/<Module>/Gen/MetaGenerated.h/.cpp`.
- Generation runs before compiling each runtime target through `nls_add_meta_generation(...)`.
- Each module emits a unique registration entry point, for example:
  - `RegisterReflectionTypes_NLS_Base`
  - `RegisterReflectionTypes_NLS_Core`
  - `RegisterReflectionTypes_NLS_Engine`
- Runtime startup code calls the generated entry point for its own module through the shared macro `NLS_META_GENERATED_REGISTER_FUNCTION`.

This avoids symbol collisions between `Base`, `Core`, `Engine`, and future runtime modules.

## What can be reflected

The current parser accepts reflection classes marked with `Meta()` under `Runtime/`.

- `Base` sample types still work.
- Engine-side types such as `Component`, `TransformComponent`, `CameraComponent`, `LightComponent`, `MeshRenderer`, `MaterialRenderer`, `SkyBoxComponent`, `GameObject`, and `Scene` are now generated by MetaParser as well.
- Inheritance from `NLS::meta::Object` is no longer required for a type to be discovered by MetaParser.

## Parser behavior

MetaParser prefers the `CppAst` path, but keeps a text fallback for headers that currently trigger `CppAst` container errors on Windows.

At the moment the fallback path is used for:

- `Runtime/Engine/Components/MeshRenderer.h`
- `Runtime/Engine/GameObject.h`
- `Runtime/Engine/SceneSystem/Scene.h`

The fallback still generates usable type and method registration so the runtime and tests stay green.

## Verification

`Tools/ReflectionTest` is the regression test for reflection registration.

It links against:

- `NLS_Base`
- `NLS_Core`
- `NLS_Engine`

And verifies:

- reflected types are present in the database
- required methods are registered
- required fields are registered
- component inheritance relationships are registered

### Recommended commands

```powershell
cmake -S . -B build
cmake --build build --config Debug --target ReflectionTest -- /m:1
D:\VSProject\Nullus\Build\bin\Debug\ReflectionTest.exe
```

Expected result:

```text
=== All reflection tests passed ===
```
